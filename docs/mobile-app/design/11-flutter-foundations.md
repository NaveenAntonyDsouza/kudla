## 11.5 Env Flavors

Three env configs — dev, staging, prod:

```dart

enum AppFlavor { dev, staging, prod }

class AppConfig {
  final String apiBaseUrl;
  final String appName;
  final bool enableDebugLogging;
  final AppFlavor flavor;

  const AppConfig({
    required this.apiBaseUrl,
    required this.appName,
    required this.enableDebugLogging,
    required this.flavor,
  });

  static const dev = AppConfig(
    apiBaseUrl: 'http://10.0.2.2:8000/api/v1',   // Android emulator → host machine
    appName: 'Kudla Matrimony [DEV]',
    enableDebugLogging: true,
    flavor: AppFlavor.dev,
  );

  static const staging = AppConfig(
    apiBaseUrl: 'https://staging.kudlamatrimony.com/api/v1',
    appName: 'Kudla Matrimony [STAGE]',
    enableDebugLogging: true,
    flavor: AppFlavor.staging,
  );

  static const prod = AppConfig(
    apiBaseUrl: 'https://kudlamatrimony.com/api/v1',
    appName: 'Kudla Matrimony',
    enableDebugLogging: false,
    flavor: AppFlavor.prod,
  );
}

// Runtime selection via --dart-define
const _flavorName = String.fromEnvironment('FLAVOR', defaultValue: 'dev');
AppConfig get appConfig => switch (_flavorName) {
  'prod' => AppConfig.prod,
  'staging' => AppConfig.staging,
  _ => AppConfig.dev,
```

## 11.6 Riverpod Pattern

One convention, applied everywhere:

```dart
// lib/features/auth/providers/auth_providers.dart

@riverpod
AuthRepository authRepository(AuthRepositoryRef ref) =>
  AuthRepository(ref.watch(apiClientProvider), ref.watch(secureStorageProvider));

@riverpod
class AuthState extends _$AuthState {
  @override
  Future<AuthStatus> build() async {
    final token = await ref.read(secureStorageProvider).readToken();
    if (token == null) return AuthStatus.unauthenticated();
    try {
      final me = await ref.read(authRepositoryProvider).fetchMe();
      return AuthStatus.authenticated(me);
    } on UnauthenticatedException {
      await ref.read(secureStorageProvider).clearToken();
      return AuthStatus.unauthenticated();
    }
  }

  Future<void> login(String email, String password) async {
    state = const AsyncLoading();
    final result = await ref.read(authRepositoryProvider).loginPassword(email, password);
    await ref.read(secureStorageProvider).writeToken(result.token);
    await ref.read(deviceRegistrarProvider).registerCurrentDevice();
    state = AsyncData(AuthStatus.authenticated(result.user));
  }

  Future<void> logout() async {
    await ref.read(authRepositoryProvider).logout();
    await ref.read(secureStorageProvider).clearToken();
    state = AsyncData(AuthStatus.unauthenticated());
  }
}
```

**Convention:** repository = Dio calls + DTO parsing. Provider = state management + side effects. UI = reads providers, renders.

## 11.7 Dio Client Setup

```dart
@riverpod
Dio apiClient(ApiClientRef ref) {
  final dio = Dio(BaseOptions(
    baseUrl: appConfig.apiBaseUrl,
    connectTimeout: const Duration(seconds: 15),
    receiveTimeout: const Duration(seconds: 15),
    contentType: 'application/json',
    responseType: ResponseType.json,
  ));

  // Auth
  dio.interceptors.add(AuthInterceptor(ref));

  // Error normalization — converts {success: false, error: {...}} into typed exceptions
  dio.interceptors.add(ErrorHandlerInterceptor());

  // Caching for site settings, reference data
  dio.interceptors.add(DioCacheInterceptor(
    options: CacheOptions(
      store: HiveCacheStore(Hive.boxName),
      policy: CachePolicy.request,
      maxStale: const Duration(hours: 24),
      priority: CachePriority.normal,
      hitCacheOnErrorExcept: [401, 403],
    ),
  ));

  // Logging (dev only)
  if (appConfig.enableDebugLogging) {
    dio.interceptors.add(LogInterceptor(responseBody: true, requestBody: true));
  }

  return dio;
}
```

### AuthInterceptor

```dart
class AuthInterceptor extends Interceptor {
  final ApiClientRef ref;
  AuthInterceptor(this.ref);

  @override
  Future<void> onRequest(RequestOptions options, RequestInterceptorHandler handler) async {
    final token = await ref.read(secureStorageProvider).readToken();
    if (token != null) options.headers['Authorization'] = 'Bearer $token';
    options.headers['Accept'] = 'application/json';
    options.headers['X-App-Version'] = '1.0.0';       // sent to /site/settings for upgrade checks
    options.headers['X-App-Platform'] = 'android';
    return handler.next(options);
  }

  @override
  Future<void> onError(DioException err, ErrorInterceptorHandler handler) async {
    if (err.response?.statusCode == 401) {
      // Token rejected — clear and bounce to login
      await ref.read(secureStorageProvider).clearToken();
      ref.read(appRouterProvider).go('/login');
    }
    return handler.next(err);
  }
}
```

### ErrorHandlerInterceptor

Maps the envelope `{success: false, error: {code, message, fields}}` into typed exceptions:

```dart
class ApiException implements Exception {
  final String code;
  final String message;
  final Map<String, List<String>>? fields;
  final int statusCode;

  ApiException(this.code, this.message, this.fields, this.statusCode);

  factory ApiException.fromResponse(Response r) {
    final err = r.data?['error'] as Map<String, dynamic>?;
    return ApiException(
      err?['code'] ?? 'UNKNOWN',
      err?['message'] ?? 'Something went wrong',
      (err?['fields'] as Map?)?.cast<String, List<String>>(),
      r.statusCode ?? 500,
    );
  }
}
```

Feature code catches `ApiException` and renders per-code UI (validation → inline field errors, DAILY_LIMIT_REACHED → toast with reset time, etc.).

## 11.9 Deep Links

### Android App Links setup

1. **Manifest** — add intent filter to launcher activity:
```xml
<intent-filter android:autoVerify="true">
  <action android:name="android.intent.action.VIEW" />
  <category android:name="android.intent.category.DEFAULT" />
  <category android:name="android.intent.category.BROWSABLE" />
  <data android:scheme="https" android:host="kudlamatrimony.com" />
</intent-filter>
```

2. **Verification** — host `/.well-known/assetlinks.json` on `https://kudlamatrimony.com/`:
```json
[{
  "relation": ["delegate_permission/common.handle_all_urls"],
  "target": { "namespace": "android_app", "package_name": "com.books.KudlaMatrimony", "sha256_cert_fingerprints": ["AB:CD:EF:..."] }
}]
```

3. **Handling** — `app_links` package streams URLs into a Riverpod provider:
```dart
@riverpod
class DeepLinkHandler extends _$DeepLinkHandler {
  @override
  void build() {
    AppLinks().uriLinkStream.listen((uri) => _handle(uri));
  }

  void _handle(Uri uri) {
    final path = uri.path;          // e.g. /profiles/AM100042
    final ref = uri.queryParameters['ref'];
    if (ref != null) _saveRefForRegistration(ref);
    ref.read(appRouterProvider).go(path);
  }
}
```

### URLs we support

| URL | Screen |
|-----|--------|
| `https://kudlamatrimony.com/` | Home (maps to `/dashboard` if logged in, else `/login`) |
| `https://kudlamatrimony.com/profile/{matriId}` | Profile view |
| `https://kudlamatrimony.com/interests/{id}` | Interest thread |
| `https://kudlamatrimony.com/reset-password/{token}` | Password reset (from email) |
| `https://kudlamatrimony.com/?ref=CODE` | Home + save affiliate code for registration |
| `https://kudlamatrimony.com/discover/{category}/{slug}` | Discover results |

## 11.10 Theme Sync with Backend

On app launch (and on resume after idle > 1h), call `GET /api/v1/site/settings`. Hydrate `SiteConfig` provider:

```dart
@Riverpod(keepAlive: true)
class SiteConfig extends _$SiteConfig {
  @override
  Future<SiteConfigData> build() async {
    final data = await ref.read(apiClientProvider).get('/site/settings');
    return SiteConfigData.fromJson(data.data['data']);
  }
}

// app.dart
@override
Widget build(BuildContext context, WidgetRef ref) {
  final config = ref.watch(siteConfigProvider);
  return config.when(
    loading: () => const SplashScreen(),
    error: (e, _) => ErrorScreen(onRetry: () => ref.invalidate(siteConfigProvider)),
    data: (data) => MaterialApp.router(
      theme: AppTheme.light(primary: data.theme.primaryColor, headingFont: data.theme.headingFont),
      routerConfig: goRouter,
    ),
  );
}
```

**Caching:** store last successful SiteConfigData in Hive. On launch, use cached value immediately + revalidate in background (stale-while-revalidate pattern). Prevents blank screen on slow networks.

**Min version gate:** `SiteConfigData.app.minimumSupportedVersion` > current build → show blocking "Update required" screen with Play Store link.

## 11.11 Secure Storage

```dart
class SecureStorage {
  final _storage = const FlutterSecureStorage(
    aOptions: AndroidOptions(encryptedSharedPreferences: true),
    iOptions: IOSOptions(accessibility: KeychainAccessibility.first_unlock),
  );

  Future<void> writeToken(String token) => _storage.write(key: 'auth_token', value: token);
  Future<String?> readToken() => _storage.read(key: 'auth_token');
  Future<void> clearToken() => _storage.delete(key: 'auth_token');

  Future<void> setBiometricEnabled(bool v) => _storage.write(key: 'biometric', value: v.toString());
  Future<bool> isBiometricEnabled() async => (await _storage.read(key: 'biometric')) == 'true';

  Future<void> clearAll() => _storage.deleteAll();
}
```

Token lives here. Biometric-unlock flag too. Everything else (dashboards, lists) goes in Hive (unencrypted, faster).

## 11.12 Splash Screen Logic

```
Splash
  ↓
Check site config cached?
  yes → apply theme; check token
  no  → fetch /site/settings; apply; check token
  ↓
Has token?
  no  → /onboarding-slides (first-time) or /login
  yes → call GET /auth/me
    200 → check onboarding_completed:
             yes → /dashboard
             no  → /register/step-{next}
    401 → clear token; /login
```

Max 3 seconds on splash (ux standard) — use cached site config + cached /me if network slow.

## 11.13 Error Screens

Shared widgets for consistent error UX:

- **Empty state** — icon + title + subtitle + optional action button. Used in empty inboxes, no results, no shortlist.
- **Error view** — icon + title + "Try again" button. Used when a fetch fails and no cached fallback.
- **Maintenance screen** — if `/site/settings` returns 503 or site.maintenance_mode=true, show this with next-available time.
- **Update required** — if current app version below `minimum_supported_version`, show blocking screen with Play Store CTA.
- **No internet** — offline toast on any network error; full screen only on launch failure.
