# Step 2 — Create `routes/api.php` Skeleton

## Goal
Replace the default `routes/api.php` generated by `install:api` with a structured skeleton that groups routes under `/api/v1/` with public vs auth sections. All routes are stubbed for now (return a placeholder); real handlers come in later steps.

## Prerequisites
- [ ] [step-01 — Install Sanctum](step-01-install-sanctum.md) complete
- [ ] `routes/api.php` exists (created by `php artisan install:api`)

## Procedure

### 1. Replace `routes/api.php`

Open `routes/api.php` and replace contents with:

```php
<?php

use Illuminate\Support\Facades\Route;

/*
|--------------------------------------------------------------------------
| API v1 Routes
|--------------------------------------------------------------------------
| All routes are prefixed with /api/v1/ (the /api/ prefix is applied
| automatically by Laravel; we add /v1/ here for versioning).
|
| Design reference: docs/mobile-app/design/01-api-foundations.md
*/

Route::prefix('v1')->group(function () {

    // ── Public (no auth) ────────────────────────────────────────
    Route::get('/health', fn () => response()->json(['success' => true, 'data' => ['status' => 'ok', 'version' => 'v1']]));

    // Public endpoints will be added in steps 6, 7, and weeks 2-4:
    // - /site/settings (step 6)
    // - /reference/{list} (step 7)
    // - /auth/register/step-1, /auth/otp/*, /auth/login/* (week 2)
    // - /membership/plans, /success-stories, /static-pages (week 3-4)
    // - /discover, /discover/{cat}, /discover/{cat}/{slug} (week 3)
    // - /webhooks/razorpay (week 4)

    // ── Auth required ───────────────────────────────────────────
    Route::middleware('auth:sanctum')->group(function () {
        Route::get('/auth/ping', fn (\Illuminate\Http\Request $r) => response()->json([
            'success' => true,
            'data' => ['user_id' => $r->user()->id, 'message' => 'authenticated']
        ]));

        // Protected endpoints will be added in weeks 2-4
    });
});
```

### 2. Verify the `/api/` prefix

Open `bootstrap/app.php` and confirm the api routing registration looks like:

```php
->withRouting(
    web: __DIR__.'/../routes/web.php',
    api: __DIR__.'/../routes/api.php',
    commands: __DIR__.'/../routes/console.php',
    health: '/up',
)
```

If `api:` line is missing, add it.

### 3. Test the routes list

```bash
php artisan route:list --path=api
```

Should show:
```
GET|HEAD   api/v1/health  …
GET|HEAD   api/v1/auth/ping  …  (auth:sanctum middleware)
```

## Verification

- [ ] `curl -H "Accept: application/json" http://localhost:8000/api/v1/health` returns `{"success":true,"data":{"status":"ok","version":"v1"}}`
- [ ] `curl -H "Accept: application/json" http://localhost:8000/api/v1/auth/ping` returns 401 (unauthenticated — no token sent)
- [ ] Create a test token via tinker:
  ```php
  $t = \App\Models\User::first()->createToken('test')->plainTextToken;
  ```
- [ ] Call ping with token:
  ```bash
  curl -H "Accept: application/json" -H "Authorization: Bearer <TOKEN>" http://localhost:8000/api/v1/auth/ping
  ```
  Returns `{"success":true,"data":{"user_id":1,"message":"authenticated"}}`.

## Common issues

| Issue | Fix |
|-------|-----|
| `GET /api/v1/health` returns 404 | Check `bootstrap/app.php` has `api:` line; run `php artisan route:clear && php artisan route:cache` |
| Protected route returns 500 not 401 | Sanctum middleware misconfigured — run `php artisan config:clear` |
| Auth works but returns web redirect HTML | `Accept: application/json` header missing (we'll fix this in step 5 with `ForceJsonResponse`) |
